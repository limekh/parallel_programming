# Лабораторная №2. Параллельное перемножение матриц(OpenMP)
## Отчёт
### Описание файлов 
- `main.cpp` - исходный код программы на C++. Реализует чтение матриц из файла, последовательное и параллельное умножение матриц, запись результата и измерение времени выполнения.
- `verify.py` - скрипт на языке Python для автоматической проверки результатов. Сравнивает результат C++ программы с расчётом через NumPy.
- `compile.bat` - компилирует `main.cpp` в `matrix_mult.exe`.
- `test.bat` - запускает тесты для разных размеров матриц (200, 400, 800, 1200) и количества потоков (1, 2, 4, 8) и собирает статистику в файл `stats.csv`.
- `input.txt` - входной файл с двумя матрицами, автоматически генерируемый при запуске тестов
- `output.txt` - выходной файл с результатом параллельного умножения. Содержит матрицу-результат, время выполнения, количество потоков, объём вычислений.
- `output_seq.txt` - выходной файл с результатом последовательного умножения. Содержит матрицу-результат, время выполнения и объём вычислений.
- `stats.csv` - файл со статистикой. Содержит данные для матриц разных конфигураций(размер, количество потоков).

### Запуск программы
#### Требования
- MinGW (g++) с поддержкой C++11
- Python 3.x с установленным пакетом `numpy`
#### Зависимости
```cmd
# Проверка g++
g++ --version

# Проверка Python
python --version

# Установка NumPy
pip install numpy
```
#### Компиляция
```cmd
compile.bat
```
#### Запуск тестов
```cmd
test.bat
```

где:  
*Seq time* - время выполнения последовательного умножения матриц(в секундах),  
*Time* - время выполнения параллельного умножения с **n** потоками(в секундах),  
*Verification* - прохождение проверки правильности умножения матриц(процесс описывается ниже) - принимает значения **PASS** в случае успеха и **FAIL** в противном случае.

### Процесс проверки
1. C++ программа умножает матрицы и сохраняет результат в output.txt
2. verify.py читает входные матрицы из input.txt
3. NumPy вычисляет результат (A × B)
4. Сравнение результата C++ с результатом NumPy
5. Вывод максимальной погрешности и статуса проверки (PASS/FAIL)

### Формат выходных данных
Так как загружаемые на Github файлы должны не превышать 25MB, загрузить `output.txt` и `output_seq.txt` не удалось. Формат этих файлов на примере `output.txt` ниже:
```
# Size: 800x800
# Threads: 4
# Time: 0.172961 s
# Operations: 1023360000
# Data: 15000 KB

800
<результат умножения - матрица 800x800>
```
