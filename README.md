# Лабораторная №4. Параллельное перемножение матриц(CUDA)
## Отчёт
### Описание файлов 
- `matrix_cuda.cu` -  Исходный код программы на CUDA C++. Реализует параллельное умножение матриц на GPU с использованием ядер `mul_sq_matrix_kernel` (глобальная память) и `mul_sq_matrix_shared_kernel` (разделяемая память).
- `verify.py` -Скрипт на языке Python для автоматической проверки результатов. Сравнивает результат CUDA программы с эталонным расчётом через NumPy.
- `plot_results.py` - Скрипт для построения графиков зависимости времени выполнения, ускорения и производительности от конфигурации блока и размера матрицы.
- `compile.bat` - Файл сборки проекта для Windows.
- `stats.csv` - Файл со статистикой. Содержит данные для матриц разных размеров и конфигураций блоков.

### Запуск программы
#### Требования
```
# Windows 10
- NVIDIA GPU с поддержкой CUDA (Compute Capability ≥ 3.0)
- CUDA Toolkit 11.0 или выше
- Visual Studio Build Tools (для компиляции)
- Python 3.x с пакетами numpy, matplotlib

# Установка Python зависимостей
pip install numpy matplotlib
```
#### Сборка
```
compile.bat
```
#### Запуск тестов
```
cd scripts
python test_cuda.py
```
#### Построение графиков
```
python plot_results.py
```

Пример работы теста для матрицы размером 800x800:
```
Matrix size: 800x800  
Block config: (16, 16)  
Memory type: global  
Kernel execution time: 12.605 ms  
Operations: 1023360000  
```

где:  
*Block config* - размер блока потоков (blockDim.x × blockDim.y),  
*Memory type* - тип используемой памяти (global/shared), 
*Kernel execution time* - время выполнения ядра на GPU (в миллисекундах),
*Operations* - количество арифметических операций (~2×N³ для умножения матриц).

### Процесс проверки
1. CUDA программа умножает матрицы на GPU и сохраняет результат в output/result.txt
2. verify.py читает входные матрицы из input/matrix.txt
3. NumPy вычисляет эталонный результат (A × B) на CPU
4. Сравнение результата CUDA с эталоном (максимальная погрешность)
5. Вывод максимальной погрешности и статуса (PASS при погрешности < 10⁻³, FAIL иначе)

### Формат выходных данных
```
# Size: 800x800
# Block Size: 16x16
# Memory Type: Global
# Time: 0.012605 s
# Operations: 1023360000
# Data: 7500 KB

800
<результат умножения - матрица 800×800>
```

### Вывод 
Распараллеливание умножения матриц с использованием CUDA на видеокарте NVIDIA GTX 1060 3GB показало эффективность для больших размеров матриц (от 800×800 и выше). Оптимальная конфигурация сетки блоков - 32x32 потока, что обеспечивает максимальную загрузку потоковых мультипроцессоров архитектуры Pascal. Использование разделяемой памяти обеспечивает дополнительное ускорение 7-10% за счёт снижения обращений к глобальной памяти.
Время выполнения растёт кубически с увеличением размера матрицы, что соответствует теоретической сложности O(n^3). Достигнутая производительность ~125 GFLOPS составляет около 2.8% от теоретического максимума видеокарты (~4.4 TFLOPS), что указывает на наличие резервов для дальнейшей оптимизации.
Сравнение с предыдущими лабораторными работами показало, что CUDA обеспечивает значительно более высокую производительность по сравнению с MPI (~125 GFLOPS против ~1-2 GFLOPS), что объясняется массивно-параллельной архитектурой GPU, оптимизированной для выполнения однотипных операций над большими массивами данных.
