# Лабораторная №3. Параллельное перемножение матриц(MPI)
## Отчёт
### Описание файлов 
- `main_mpi.cpp` - Исходный код программы на C++ с использованием MPI. Реализует распределённое умножение матриц с использованием `MPI_Bcast`, `MPI_Scatterv`, `MPI_Gatherv`.
- `verify.py` - Cкрипт на языке Python для автоматической проверки результатов. Сравнивает результат C++ программы с расчётом через NumPy.
- `run_test.sh` - Bash-скрипт для Linux. Запускает тесты для разных размеров матриц (200, 400, 800, 1200) и количества процессов (1, 2, 4, 6).
- `plot_results.py` - Скрипт для построения графиков зависимости времени и ускорения от количества процессов.
- `Makefile` - Файл сборки проекта. Компилирует программу с флагами MPI.
- `input/matrix.txt` - Входной файл с двумя матрицами. Генерируется автоматически при запуске тестов.
- `output/result.txt` - Выходной файл с результатом умножения. Содержит матрицу-результат, время выполнения, количество процессов, объём вычислений.
- `stats.csv` - Файл со статистикой. Содержит данные для матриц разных конфигураций(размер, количество потоков).

### Запуск программы
#### Требования
```
# Ubuntu/Debian
sudo apt update
sudo apt install -y openmpi-bin libopenmpi-dev python3 python3-pip
pip3 install numpy matplotlib
```
#### Сборка
```bash
make all
```
#### Запуск тестов
```bash
make test
```
#### Построение графиков
```bash
python3 plot_results.py
```
#### Полное тестирование 
```bash
make full-test
```

### Процесс проверки
1. MPI программа умножает матрицы и сохраняет результат в output/result.txt
2. verify.py читает входные матрицы из input/matrix.txt
3. NumPy вычисляет эталонный результат (A × B)
4. Сравнение результата MPI с эталоном
5. Вывод максимальной погрешности и статуса (PASS/FAIL)

### Вывод 
Третья лабораторная работа продемонстрировала возможности технологии MPI для распараллеливания вычислительно сложных задач. Несмотря на более высокие накладные расходы по сравнению с OpenMP, MPI обеспечивает лучшую масштабируемость и применимость в распределённых системах. Оптимальная эффективность достигается при работе с большими матрицами, где вычислительная нагрузка компенсирует затраты на межпроцессную коммуникацию.
